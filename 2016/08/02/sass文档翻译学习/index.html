Sass（极好的语义化层叠样式表）

Sass是一种CSS扩展，它在基本语言上添加了更强大和优雅的编程方式。它允许你使用变量，嵌套规则，mixins，行内import，以及更多，而且全面兼容CSS语法。Sass帮忙保持大量的层叠样式被很好的组织，而且使得层叠样式更小并更快地运行，尤其使用罗盘样式库。

特性

  • 完全兼容CSS
  • 语言扩展，例如变量，嵌套和mixin
  • 许多有用的方法来操作颜色和其它变量
  • 高级特性，像对库的控制指令
  • 结构良好的，可自定义的输出

语法

Sass有两个可用的语法。第一个，被称为SCSS，并被用在这篇参考文档中，是CSS语法的一个扩展。这意味着每一个有效的CSS文件都是一个有效的SCSS文件，有着相同的意义。另外，SCSS明白大部分的CSS的hacks和供应商语法，像IE的filter语法。这个语法是下面介绍的Sass语法的加强。使用这个语法的文件的扩展名为.scss。

第二个和更老的语法，被称为缩进语法，提供了一个更简明的写CSS的方式。它使用缩进而不是大括号来指定选择器的缩进，使用新行而不是分号来分隔特性。一些人发现这种语法比SCSS更容易阅读和更快编写。缩进语法拥有所有相同的特性，尽管其中有一些不同的语法。使用这个语法的文件扩展名为.sass

每个语法都能从其它import文件进来。文件能被自动转换从一个语法到另一个语法使用sass-convert命令行工具。

注意这个命令不能产生CSS文件，而要使用sass命令来做这个

SassScript

除了一般的CSS特性语法，Sass支持一些扩展，称为SassScript。SassScript允许特性使用变量，算术，和额外函数。SassScript能被用在任意的特性中。

SassScript能被用来产生选择器和特性名，在写mixin的时候非常有用。它是通过interpolation来实现的。

数据类型

SassScript支持7中主要的数据类型

  • numbers， 1.2, 13, 10px
  • strings of text，有或者没有引号
  • colors，blue，#000000
  • booleans，true, false
  • nulls，null
  • list of values，用空格或者逗号分隔，1.5em 1em 0 2em, Helvetica, Arial, sans-serif
  • maps from one value to another,(key1: value1, key2: value2)

SassScript也支持所有其它的CSS特性值，像Unicode ranges和!important声明。然而并没有什么特殊处理，它们将会被看成无引号的字符串。

字符串

CSS指定了两种类型的字符串：有引号的，例如”Lucla”,”http://www.baidu.com"，和没有引号的，例如sans-serif和bold。SassScript识别两种类型。

Lists

Lists是Sass展现这样的CSS声明margin:10px 15px 0 0。Lists就是一系列其它类型的值，由空格或逗号分隔。事实上，单个值也被视为Lists。

就其本身而言，Lists没有什么东西，但是SassScript list functions使它变得有用。nth函数能够访问Lists中的一个条目，join函数能将多个Lists合并起来。append函数能添加条目到列表中。@each指令能对每一个条目添加样式。

除了包括简单的值，Lists也能包括其它Lists。例如1px 2px, 4px 5px就是一个包含了Lists的Lists。如果内部的Lists与外部Lists有相同的分隔符，你需要使用括号来分隔。例如(1px 2px) (4px 5px)。

当Lists转换为CSS，并不会将括号添加进去，因为CSS不能理解括号。这就意味着(1px 2px) (4px 5px)和1px 2px 4px 5px将在CSS中显示一样。

Lists也可以一条条目都不包括。它们不能直接输出到CSS，如果你试图这么做了，Sass将会报错。如果列表包括空的列表或者空的值，那么这些空的值会在转换为CSS时被移除。

Maps

Maps展现了键值对的关系，键用来查找值。它将值集合进命名组中来动态访问。它们不直接与CSS类似，尽管它们与媒体查询表达式语法类似。

不同于Lists，Maps必须由括号包裹起来，并且必须由逗号分隔。

与Lists一样，Maps大部分由SassScript函数操作。map-get函数查阅Maps里的值，map-merge函数添加值到Maps中。@each指令能添加样式到每一个Maps中的键值对。键值对的顺序与创建时的顺序相同。

Maps能用在任何Lists能用的地方。当被一个Lists函数使用时，Maps被当做键值对的列表。

Maps不能被转换为CSS。

Colors

任意的CSS颜色表达式都返回一个SassScript Color值。这其中包括一大部分颜色的命名。

在压缩输出模式下，Sass将输出最小的CSS颜色表达。例如，#FF0000在压缩输出模式下将输出red，但是blanchedalmond将输出#FFEBCD。

操作

所有的类型都支持相等操作符== !=，每一个操作符都有其特殊的操作符支持。

数字操作

SassScript支持标准的算术操作符。Sass数学函数在进行算术操作中保存单位。这意味着，你不能够在数字上使用不兼容的单位，而且两个有相同单位的数字在相乘的情况下将产生平方的单位。意识到px * px是一个无效的CSS单位，你将得到一个错误。

关系操作符(< > <= >=)在Number类型下支持。(== !=)在所有类型下支持。

CSS允许使用/在特性中分隔数字。SassScript作为CSS的扩展，它必须支持这个，然而也支持将其作为除号。这意味着默认情况下，如果两个数字被/分隔，它将在CSS中显示得一样。
