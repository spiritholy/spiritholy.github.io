基于vue-cli的webpack模板构建的一个多页应用模板

发表于 2017-03-03   |  

vue-cli提供的webpack模板，默认是一个单页应用的模板，它只有一个入口文件，构建后也是生成一个html文件，但我可能会遇到项目并不是一个单页应用，比如一个网站，比如一个博客，但是我也希望利用vue的组件化开发，而不想写重复的代码。

公司最近要将一个之前的一个管理系统添加许多新页面和功能，我打算用vue来完成这些新的页面，利用vue项目的工程结构，更加能使得代码可维护和可重用。但是系统之前都是一个页面一个html，并不是单页应用，所以为了与之前保持一致，我需要一个能生成多页应用的vue模板。

源码请查看https://github.com/spiritholy/vue2-multipage-template

还有什么问题

在webpack的配置中由于使用了CommonsChunkPlugin插件，可以把公共的js提取到vendor.js中，这样最后打包出来的vendor.js所有html都可以引用。

但是我发现如果所有页面都使用了同一个css样式库的话，比如bootstrap，最后打包出来的css文件中每一个文件都包含一遍bootstrap代码。这样其实是不好的，应该让每一个html都引用一个相同的bootstrap的文件，这样可以使浏览器使用缓存功能，减少页面加载的时间。
